Applied to build scripts and `@CompileStatic` convention sources:

- **`list` kind — operator mutations and removed setter** (Cat-C / Cat-A): `+=` / `<<` on lazy `ListProperty` values rewritten to `.add(...)` / `.addAll(...)`; the removed `JavaExec.setArgs(...)` rewritten to `getArgs().set(...)`:

  - `build-logic/.../org.apache.groovy-core.gradle` — `AntlrTask.arguments +=` → `.addAll(...)`
  - `build-logic/.../org.apache.groovy-internal.gradle` — `CompileOptions.compilerArgs <<` → `.add(...)`, `GroovyCompile.groovyOptions.forkOptions.jvmArgs +=` → `.addAll(...)`
  - `subprojects/groovy-console/build.gradle` — `JavaExec.jvmArgs +=` → `.addAll(...)`
  - `subprojects/tests-preview/build.gradle` — `Test.jvmArgs +=` → `.add(...)`
  - `build-logic/.../PerformanceTestsExtension.groovy` — `JavaExec.setArgs(args)` → `args.set(args)`; an adjacent read `je.args.asList()` (now a `ListProperty`) → `je.args.get()`; and a `@CompileStatic` assignment `je.jvmArgs = [...]` → `je.jvmArgs.set([...])` (lazy-assignment DSL doesn't apply under static compilation).

### Manual fixes beyond the migration data (build-error driven, tasks 07–08)

- **Self-referencing `ConfigurableFileCollection`s** (`file_collection` kind, circular-ref): the preview rejects assignments where the new value references the same collection.
  - `build-logic/.../org.apache.groovy-tested.gradle` and `subprojects/groovy-sql/build.gradle` — `Test.classpath = … + classpath` rewritten to append the extra test source dirs with `classpath.from(...)`.
  - `build-logic/.../org.apache.groovy-core.gradle` — the bootstrap `GroovyCompile.groovyClasspath`/`classpath = files(bootstrapJar, classpath)` rewritten with `setFrom(files(bootstrapJar, <prop>.files))` to preserve bootstrap-compiler precedence (order-sensitive).
- **`DirectoryProperty` read** — `AntlrTask.outputDirectory.canonicalPath` → `outputDirectory.get().asFile.canonicalPath`.
- **`ConfigurableFileCollection` compound assignment** — `Javadoc.classpath += project.file(...)` → `classpath.from(...)`.
- **Javadoc multi-value option** — `Javadoc` `addStringOption('tag', …)` (called 3×) replaced with `addMultilineStringsOption('tag').setValue([...])`; the preview treats `tag` as a `List`-typed option, so setting a single `String` failed.
